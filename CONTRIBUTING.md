# Contributing to Wabi Saby

Thanks for your interest in contributing. Each repository in the [Wabi Saby](https://github.com/wabisaby) organization may have its own setup; this is the default guide.

## Where to contribute

- **Platform (core, API, plugin runtime):** [wabisaby-core](https://github.com/wabisaby/wabisaby-core)
- **Plugin SDK:** [wabisaby-plugin-sdk-go](https://github.com/wabisaby/wabisaby-plugin-sdk-go)
- **Plugins (sources, storage, etc.):** [wabisaby-plugins](https://github.com/wabisaby/wabisaby-plugins)
- **DevKit (desktop app, meta-repo):** [wabisaby-devkit](https://github.com/wabisaby/wabisaby-devkit)
- **Protos:** [wabisaby-protos](https://github.com/wabisaby/wabisaby-protos)

Check the specific repo’s README and any local `CONTRIBUTING.md` for build, test, and submission steps.

## How to contribute

1. **Open an issue** — Describe the bug, feature, or change so we can align before you invest time.
2. **Fork and branch** — Create a branch from the default branch (e.g. `main`).
3. **Make your changes** — Follow the project’s style (formatting, tests, docs).
4. **Open a pull request** — Reference the issue and keep the PR focused. Maintainers will review and may request changes.

## Code and conduct

- Follow the [Code of Conduct](CODE_OF_CONDUCT.md).
- Keep discussions constructive and on-topic.

## Questions

Open a [Discussion](https://github.com/orgs/wabisaby/discussions) or an issue in the relevant repository.
